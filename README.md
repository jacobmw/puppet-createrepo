## Overview

`createrepo` creates and updates yum repositories. It defines a single
Puppet `define` type that manages the repository and cache directories,
installs the `createrepo` (or `createrepo_c`) package, renders a metadata
update shell script, and optionally schedules updates via cron and/or
applies them during the Puppet agent run.

It does **not** manage the directory tree above the repository root and
makes no attempt to manage an HTTP server for repository clients.

## Differences from upstream

This fork (v4.x) is a Puppet 8 modernization of the original
`palli/createrepo` module. Notable changes:

- **Puppet 8 compatible.** Module requirement bumped to
  `>= 7.24 < 9.0.0`.
- **Typed parameters.** All parameters now use the Puppet 4+ type system
  (`Stdlib::Absolutepath`, `Boolean`, `Integer`, `Stdlib::Filemode`, …).
  Invalid parameter values raise compile-time type errors instead of the
  legacy stdlib `validate_*` messages.
- **Modern fact API.** OS detection uses `$facts['os']['family']` instead
  of `$::osfamily`.
- **Dependencies bumped.** Requires `puppetlabs/stdlib >= 9` and
  `puppetlabs/cron_core >= 1` (the `cron` core type is no longer bundled
  with Puppet 8).
- **Supported OS matrix refreshed.** RHEL/Rocky 8 & 9 and Debian 11 & 12.
  Support for EL5/6/7, Debian 7/8 and Ubuntu 12.04/14.04 has been
  dropped.
- **Acceptance testing migrated to [Litmus][litmus].** Beaker nodesets
  and `.travis.yml` have been removed; CI runs via GitHub Actions
  (`.github/workflows/20-pdk.yml`).
- **Docs generated by puppet-strings.** The manifest has been
  re-commented to play well with `puppet strings generate`.

The v3 behavioral change introduced upstream is preserved: setting
`enable_cron => false` does **not** imply "update during the Puppet
agent run" — enable that explicitly with `enable_update => true`.

[litmus]: https://github.com/puppetlabs/puppet_litmus

## Module Description

Yum repositories are a distribution method for RPM packages, usually
served via HTTP. This module creates yum repositories and their
metadata caches, and provides two mechanisms to keep them current:

- a cron job (default: every 10 minutes), and
- an optional in-run `Exec` triggered on every Puppet agent run.

## Setup

### What createrepo affects

- repository and cache directories
- the `createrepo` (or `createrepo_c`) package
- cron jobs for repo updates (optional)
- an update script, placed in `/usr/local/bin` by default

### Beginning with createrepo

```puppet
createrepo { 'yumrepo':
  repository_dir => '/var/yumrepos/yumrepo',
  repo_cache_dir => '/var/cache/yumrepos/yumrepo',
}
```

## Usage

The module provides a single `define`, so as many repositories can be
created as needed — typically at least a `stable` and `testing` repo.

### Switching to `createrepo_c`

`createrepo_c` is the modern C reimplementation; on EL 8 and 9 it is the
recommended package. To use it, set both `createrepo_package` and
`createrepo_cmd` — there is no auto-derivation:

```puppet
createrepo { 'yumrepo':
  createrepo_package => 'createrepo_c',
  createrepo_cmd     => '/usr/bin/createrepo_c',
}
```

### Regarding checksums

Older versions of yum do not support some later default checksum types.
From the `createrepo` man page:

> Choose the checksum type used in repomd.xml and for packages in the
> metadata. The default is now "sha256" (if python has hashlib). The
> older default was "sha", which is actually "sha1"; however explicitly
> using "sha1" doesn't work on older (3.0.x) versions of yum — you need
> to specify "sha".

Use the `checksum_type` parameter to override the default. This option
is RedHat-family only.

### Parameters

| Parameter              | Type                              | Default                                              |
| ---------------------- | --------------------------------- | ---------------------------------------------------- |
| `repository_dir`       | `Stdlib::Absolutepath`            | `/var/yumrepos/${name}`                              |
| `repo_cache_dir`       | `Stdlib::Absolutepath`            | `/var/cache/yumrepos/${name}`                        |
| `repo_owner`           | `String[1]`                       | `root`                                               |
| `repo_group`           | `String[1]`                       | `root`                                               |
| `repo_mode`            | `Stdlib::Filemode`                | `0775`                                               |
| `repo_recurse`         | `Boolean`                         | `false`                                              |
| `repo_ignore`          | `Optional[Array[String[1]]]`      | `undef`                                              |
| `repo_seltype`         | `String[1]`                       | `httpd_sys_content_t`                                |
| `enable_cron`          | `Boolean`                         | `true`                                               |
| `enable_update`        | `Boolean`                         | `false`                                              |
| `cron_minute`          | `String[1]`                       | `*/10`                                               |
| `cron_hour`            | `String[1]`                       | `*`                                                  |
| `cron_weekday`         | `String[1]`                       | `*`                                                  |
| `changelog_limit`      | `Integer[0]`                      | `5` (RedHat only)                                    |
| `checksum_type`        | `Optional[String[1]]`             | `undef` (RedHat only)                                |
| `update_file_path`     | `Optional[Stdlib::Absolutepath]`  | `/usr/local/bin/createrepo-update-${name}`           |
| `suppress_cron_stdout` | `Boolean`                         | `false`                                              |
| `suppress_cron_stderr` | `Boolean`                         | `false`                                              |
| `groupfile`            | `Optional[String[1]]`             | `undef`                                              |
| `workers`              | `Optional[Integer[1]]`            | `undef`                                              |
| `timeout`              | `Integer[0]`                      | `300`                                                |
| `manage_repo_dirs`     | `Boolean`                         | `true`                                               |
| `cleanup`              | `Boolean`                         | `false`                                              |
| `cleanup_keep`         | `Integer[0]`                      | `2`                                                  |
| `use_lockfile`         | `Boolean`                         | `false`                                              |
| `lockfile`             | `Stdlib::Absolutepath`            | `/tmp/createrepo-update-${name}.lock`                |
| `createrepo_package`   | `String[1]`                       | `createrepo`                                         |
| `createrepo_cmd`       | `Stdlib::Absolutepath`            | `/usr/bin/createrepo`                                |

See the puppet-strings docblock at the top of `manifests/init.pp` (or
run `puppet strings generate`) for full per-parameter descriptions.

## Limitations

- Supported operating systems (Litmus-tested): RHEL & Rocky Linux **8**
  and **9**; Debian **11** and **12**.
- Supported Puppet versions: **7.24+** through **8.x**.
- `$createrepo_package` and `$createrepo_cmd` are *not* coupled — when
  you switch to `createrepo_c`, set both.
- `$name` is sanitized (`/` → `-`) only when computing the default
  `update_file_path`. If you pass `update_file_path` yourself, no
  sanitization is performed.
- The `Exec["createrepo-${name}"]` resource uses
  `creates => "${repository_dir}/repodata"` as its idempotency guard, so
  the create-time exec will not re-run once the repodata directory
  exists. Ongoing updates are the cron job's / script's responsibility.
- Non-RHEL platforms: the createrepo shipped with Debian doesn't support
  `--changelog-limit` or `--checksum`, so those parameters are silently
  ignored there.

## Development

1. Fork the repo.
2. Run the tests (see below). PRs need passing tests.
3. Add a test for your change. Only refactoring and docs changes are
   exempt.
4. Make the test pass.
5. Push to your fork and submit a pull request.

See [`CONTRIBUTING.md`](CONTRIBUTING.md) for prerequisites and the
detailed workflow.

### Running tests

This project uses [PDK][pdk] for validation and unit tests, and
[Litmus][litmus] for acceptance tests.

```sh
# Validation (puppet syntax, metadata, lint, rubocop)
pdk validate

# Unit tests (all)
pdk test unit

# Single unit spec
pdk test unit --tests=spec/defines/createrpo_spec.rb

# Acceptance via Litmus (RHEL/Rocky 8 & 9 by default)
bundle exec rake 'litmus:provision_list[default]'
bundle exec rake 'litmus:install_agent'
bundle exec rake 'litmus:install_module'
bundle exec rake 'litmus:acceptance:parallel'
bundle exec rake 'litmus:tear_down'
```

[pdk]: https://www.puppet.com/docs/pdk/latest/pdk.html
